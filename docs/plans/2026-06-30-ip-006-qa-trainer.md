# IP-006 Flashcard and Q&A Trainer Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add interview flashcards and spoken-answer practice with rubric scoring.

**Architecture:** Store flashcards as typed static data linked to technologies and topics. Record answer attempts in app state and feed scores into the revision scheduler.

**Tech Stack:** React, TypeScript, Vitest, React Testing Library.

---

## Ticket

**Epic:** E3 Practice Tools

**User value:** The learner can practice concise interview answers and turn weak answers into revision work.

## Acceptance Criteria

- Practice view has a Q&A mode with flashcards by technology/topic.
- A card shows prompt, expected answer outline, follow-up prompt, and rubric.
- The learner can mark an answer score for accuracy, concision, trade-off awareness, and example quality.
- Submitting a score creates an attempt that affects revision/progress state.
- Tests cover card filtering and scoring.
