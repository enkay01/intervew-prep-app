# IP-001 App Shell and Navigation Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Create the smallest local React/TypeScript app shell with navigation, responsive layout, and placeholder views for the study workflow.

**Architecture:** Use Vite with React and TypeScript. Keep all state local to placeholder components until later tickets add domain data and persistence. Use plain CSS in one global stylesheet.

**Tech Stack:** Vite, React, TypeScript, Vitest, React Testing Library, plain CSS.

---

## Ticket

**Epic:** E1 Local App Foundation and Curriculum Browser

**User value:** The learner can run the app locally and see the future study areas without reading docs.

## Acceptance Criteria

- `npm install`, `npm run dev`, `npm run build`, and `npm test` work.
- The app renders a header, primary navigation, and views for Dashboard, Resources, Practice, Mock Interview, and Settings.
- Navigation changes the visible view without a page refresh.
- The layout works at 390px wide and desktop width.
- The page contains no auth, account, team, payment, or hosted database UI.

## Plan

### Task 1: Scaffold the app

**Files:**
- Create: `package.json`
- Create: `index.html`
- Create: `src/main.tsx`
- Create: `src/App.tsx`
- Create: `src/styles.css`
- Create: `src/test/setup.ts`
- Create: `vite.config.ts`
- Create: `tsconfig.json`
- Create: `tsconfig.node.json`

**Steps:**

1. Add Vite, React, TypeScript, Vitest, jsdom, and React Testing Library dependencies.
2. Add scripts: `dev`, `build`, `test`, and `preview`.
3. Create the Vite HTML entry and React mount point.
4. Create `App.tsx` with in-memory selected view state.
5. Create accessible nav buttons using `aria-current="page"` for the active view.
6. Add global CSS with stable spacing, readable contrast, and responsive nav wrapping.

### Task 2: Add smoke tests

**Files:**
- Create: `src/App.test.tsx`

**Steps:**

1. Write a test that renders the app and finds the main heading.
2. Write a test that clicks Resources and verifies the resources view appears.
3. Run `npm test -- --run`.
4. Fix the app until tests pass.
