# July Interview Readiness Plan

## Time budget

Because the goal is to be interviewing within July, use an intensive but sustainable schedule:

- **Weekdays:** 4 hours/day.
- **Saturdays:** 5-6 hours for timed builds and mock interviews.
- **Sundays:** 2-3 hours for revision, planning, and recovery.
- **Minimum viable day:** 90 minutes when work/life intervenes: 30 minutes revision, 45 minutes coding, 15 minutes Q&A.

## Daily structure

1. **Revision, 30-45 minutes:** revisit yesterday's mistakes and flashcards due today.
2. **Concept learning, 60-75 minutes:** official docs or focused lesson.
3. **Build exercise, 90-120 minutes:** implement a feature or coding task.
4. **Interview output, 30 minutes:** answer Q&A aloud and write a concise model answer.
5. **Coding screen practice, 30-45 minutes:** data structures/algorithms or timed TypeScript utility.

## Weekly focus

### Week 1: React and TypeScript foundation

- React components, props, state, events, lists, forms, hooks, effects, and custom hooks.
- TypeScript everyday types, narrowing, functions, object types, unions, and React props/events.
- Practice builds: reusable component library, form flow, and dashboard widgets written by the learner as exercises.
- Mock: React Q&A plus one timed component exercise.

### Week 2: Next.js full-stack patterns

- Routing, layouts, server/client components, data fetching, caching, forms, route handlers, and deployment.
- Practice builds: pages, forms, route handlers, attempts, and persistence patterns written by the learner as exercises.
- Mock: explain rendering strategies and debug hydration/data fetching scenarios.

### Week 3: Python APIs with FastAPI and Flask

- FastAPI validation, routers, dependencies, auth, async, testing, and OpenAPI.
- Flask routes, blueprints, app factory, request context, config, and testing.
- Practice builds: exercise API in FastAPI and comparison lab in Flask written by the learner.
- Mock: design an API contract and compare FastAPI vs Flask.

### Week 4: React Native, integration, and interviews

- React Native components, styling, FlatList, TextInput, navigation, storage, and platform differences.
- Practice build: small Expo prototype or mobile-first progress view written by the learner.
- Integration: full mock interviews, portfolio story, STAR examples, and weak-area remediation.

## Daily plan template

| Day type | Tasks |
| --- | --- |
| New concept day | Review 5 flashcards, read official docs section, implement one feature, write 5 Q&A answers, solve one coding problem. |
| Build day | Review previous bugs, build a vertical slice, test it, document trade-offs, record a 2-minute explanation. |
| Mock day | 30-60 minute mock interview, score rubric, identify top 3 gaps, schedule remediation. |
| Recovery/revision day | Rework failed exercises, update flashcards, summarize patterns, light coding problem. |

## Coding test cadence

- 4 algorithm problems/week from arrays, hash maps, two pointers, stacks, binary search, sliding window, trees, and graphs.
- 3 frontend timed tasks/week: component, form, state management, data fetching, or bug fix.
- 2 backend timed tasks/week: endpoint, validation, persistence, auth, or tests.
- 1 system/API design prompt/week.

## Readiness gates

Before applying broadly, be able to:

- Build a typed React form with validation without notes.
- Explain useEffect dependencies and avoid common effect mistakes.
- Build a Next.js page with server-side data and a client interactive component.
- Type API responses and component props safely in TypeScript.
- Build and test a FastAPI endpoint with validation.
- Explain Flask app structure and when Flask is preferable to FastAPI.
- Complete a 45-minute coding screen with a clear verbal explanation.
- Explain one end-to-end project architecture confidently.
